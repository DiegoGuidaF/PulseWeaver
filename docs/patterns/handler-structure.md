# Handler Structure

Handlers are the HTTP boundary layer. They translate between OpenAPI DTOs and domain types — nothing more.

## Pattern

```
HTTP Handler → extract primitives from DTO → call Service → map result to response
```

- Handlers live in `internal/<domain>/handler.go`
- Each handler struct holds a reference to its service (and a logger)
- Handlers implement methods on `StrictServerInterface` (generated by oapi-codegen)
- **Never pass generated `httpapi` types deeper than the handler** — extract primitives and pass domain types to services

## Scaffold

```go
type HTTPHandler struct {
    service *Service
    logger  *slog.Logger
}

func NewHTTPHandler(service *Service, logger *slog.Logger) *HTTPHandler {
    return &HTTPHandler{service: service, logger: logger}
}

func (h *HTTPHandler) CreateDevice(ctx context.Context, req httpapi.CreateDeviceRequestObject) (httpapi.CreateDeviceResponseObject, error) {
    logger := logging.FromCtx(ctx)
    logger.Info("creating device")

    // Extract primitives from the DTO
    device, err := h.service.CreateDevice(ctx, CreateDeviceInput{
        Name: req.Body.Name,
    })
    if err != nil {
        // Map domain errors to HTTP responses
        if errors.Is(err, ErrDeviceNameConflict) {
            return httpapi.CreateDevice409JSONResponse{...}, nil
        }
        return nil, err
    }

    // Map domain type to response DTO
    return httpapi.CreateDevice201JSONResponse{
        Id:   device.ID,
        Name: device.Name,
    }, nil
}
```

## Key rules

- **Error mapping**: match domain sentinel errors (`ErrNotFound`, `ErrConflict`) to HTTP status codes. Return `nil, err` only for unexpected errors (becomes 500).
  - `ErrBadRequest` (from a Params constructor) → 400. Use `errors.Is(err, ErrBadRequest)` and return the error message directly — it is already user-readable. See service-layer.md for the Params pattern.
- **Logging**: call `logging.FromCtx(ctx)` to get the request-scoped logger. Set the `operation` attribute at handler entry.
- **No business logic**: validation beyond "does this field exist" belongs in the service.
- **Auth**: authentication is handled by middleware (`PrincipalUserContextMiddleware` / `PrincipalDeviceContextMiddleware`). Handlers retrieve the principal from context when needed.
